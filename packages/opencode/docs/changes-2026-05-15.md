## 三、关键技术决策

### 1. octo_canva → octo_studio 统一命名

UXAI 中 agent 名为 `studio`，而 octoAI 中原有 `octo_canva`。为保持命名一致性（`octo_` 前缀），统一命名为 `octo_studio`。所有引用点（agent 定义、session category、skills 页面、前端 session 创建）都已同步更新。

### 2. Studio 前端 lazy import 路径

删除旧 `pages/studio.tsx` 后，改为 `pages/studio/index.tsx` 目录结构。路由文件中的 lazy import 路径必须显式指定为 `@/pages/studio/index`，避免模块解析歧义。

### 3. Agent Prompt 合并策略

原 `octo_canva.txt` 内容聚焦 MCP 服务协调，UXAI `studio.txt` 聚焦 Jimeng/Internal 工具调用。最终采用 UXAI 版本并追加 `creative-assets` skill 引用行，兼顾两种场景。

### 4. Skills 配置路径修正

`desktop/src/main/ipc.ts` 中 Skills 配置文件路径从 `~/.config/octo/skills.json` 改为 `app.getPath("userData")/skills.json`，符合 Windows/macOS 各平台的应用数据目录规范。
