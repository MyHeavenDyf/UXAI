## 二、提交记录

### Commit 1: `5ca0babb0` — Octo Agent provider 无 API Key 时抛出友好认证错误提示

**问题**: Octo Agent provider 在 `autoload` 阶段无条件返回 `true`，即使没有 API Key。`resolveSDK` 中 key 解析完成后，无 Key 时静默 401 失败，用户得不到明确提示。

**修改内容**:

| 文件 | 改动 |
|------|------|
| `src/provider/provider.ts` | 导入 `AuthError`，在 `resolveSDK` 的 key 解析后添加 opencode provider 无 Key 检查 |

**关键决策**: 保持 `autoload: true` 不变。改为 `autoload: false` 会导致首次用户在 UI 中完全看不到 Octo Agent provider，无法进入输入 API Key 的流程。因为 opencode 是全新 provider，没有 env/auth 阶段数据，`providers["opencode"]` 在 custom loader 时为 `undefined`。

---

### Commit 2: `c55ba24ad` — 修复 skills.json 读取在 Electron/Node 环境下的 Bun is not defined 错误

**问题**: 桌面端启动后 agent 接口返回 500，错误为 `ReferenceError: Bun is not defined`。根因是 `skill/index.ts` 中使用 `Bun.file()` 读取 skills.json 配置，但 Electron 运行在 Node.js 环境。

**修改内容**:

| 文件 | 改动 |
|------|------|
| `src/skill/index.ts` | 将 `Bun.file().json()` 替换为 `fs/promises` 的 `readFile` + `JSON.parse`，使用 `Effect.tryPromise` + `Effect.catch` 保持错误处理兼容 |

---

### Commit 3: `28f2e6be3` — 修复 Electron/Node 环境兼容性：import.meta.dir 和 drizzle migrate 条件导入

**问题**: 全面排查 Bun API 兼容性后，发现两个高风险隐患：

1. `skill/index.ts` 的 `import.meta.dir` 是 Bun 专有属性，Node.js 使用 `import.meta.dirname`
2. `storage/db.ts` 直接导入 `drizzle-orm/bun-sqlite/migrator`，未通过 `#db` 条件导入隔离

**修改内容**:

| 文件 | 改动 |
|------|------|
| `src/skill/index.ts` | `import.meta.dirname`(Node 标准)优先，fallback 到 `import.meta.dir`(Bun) |
| `src/storage/db.ts` | 移除 `drizzle-orm/bun-sqlite` 直接导入，改为从 `#db` 条件导入 `migrate`；`Client` 类型改为 `ReturnType<typeof init>` |
| `src/storage/db.bun.ts` | 新增导出 `drizzle-orm/bun-sqlite/migrator` 的 `migrate` |
| `src/storage/db.node.ts` | 新增导出 `drizzle-orm/node-sqlite/migrator` 的 `migrate` |

**注**: `storage/db.ts` 的 `drizzle-orm/bun-sqlite` 直接导入是上游原有设计，非近期修改引入。此次修复为预防性改进。

## 三、Octo Agent Provider Bug 分析

### Bug 1: autoload 无条件为 true（已修复）

**位置**: `src/provider/provider.ts:160-218`

**现象**: `ok` 变量被计算但未使用，`autoload` 始终为 `true`。

**修复方案**: 保持 `autoload: true`，在 `resolveSDK` 中添加无 Key 时的 `ProviderAuthError` 友好提示。

**API Key 存储路径**:
- `~/.config/octo/octo.jsonc` — 优先写入位置
- `~/.config/opencode/octo.jsonc` — 回退位置

**Key 解析链**: `OPENCODE_API_KEY` env → auth store → config `provider.opencode.options.apiKey`

### Bug 2: Agent Skills 未按 agent 限定范围（已记录，待修复）

**状态**: 记录在 CLAUDE.md 已知问题中

**现象**: `agent.skills` 字段已声明但 `Skill.available()` 只按 permission 过滤，所有 agent 能看到全部 skill。
