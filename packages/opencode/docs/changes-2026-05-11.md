## 二、提交记录

### Commit 1: `8d89a8aae` — 还原重命名并恢复功能

**问题**: 之前两次 commit（`c3b80dc0e`、`2a8b36090`）将 `OPENCODE_*` flags、`x-opencode-*` headers 等重命名为 `OCTO_*` / `x-octo-*`，导致 SDK 与服务器通信失败。

**修改内容**:

| 类别 | 改动 | 文件数 |
|------|------|--------|
| Flag 还原 | `OCTO_*` → `OPENCODE_*` | 93 |
| Header 还原 | `x-octo-*` → `x-opencode-*` | 8 |
| 测试/脚本还原 | 测试和脚本中的 flag 引用 | 33 |
| 主 Agent 重命名 | `build` → `octo_ai` | 1 |
| 4 个专业 Agent 恢复 | octo_insight/make/design/canva 定义 + prompt | 1 |
| Skills 字段 | Agent.Info 新增 skills 字段 | 2 |
| 内置 Skill 发现 | 扫描 `src/agent/skills/` 目录 | 1 |
| 配置双读取 | 同时支持 `.opencode` 和 `.octo` 配置文件 | 1 |
| Agent 配置 schema | config/agent.ts 添加 skills 字段 | 1 |

**新增 Agent 定义**:
- `octo_insight` — 用户访谈分析专家（skills: interview-analysis）
- `octo_make` — Web 设计原型专家（skills: html-prototype）
- `octo_design` — UI 设计专家（skills: design-basics）
- `octo_canva` — 创意素材生成专家（skills: creative-assets）

**向后兼容**:
- `get("build")` 自动映射到 `octo_ai`
- `default_agent: "build"` 自动解析为 `octo_ai`
- 配置中 `"build"` 键自动映射到 `"octo_ai"`
